# Codegraph — code intelligence over an indexed knowledge graph

Codegraph is a SQLite knowledge graph of every symbol, edge, and file in
the workspace — pre-computed structure you would otherwise re-derive by
reading files (cached intelligence: thousands of parse/trace decisions you

blast radius in view. More accurate context, in far fewer tokens and
round-trips than reading files yourself.

[assistant] Gin's request routing and middleware execution work through a single unified handler chain. Here's the full path:

## 1. Middleware registration builds a combined chain (at setup time)

When you register routes, gin doesn't keep middleware separate from handlers — it flattens everything into one `HandlersChain` (a `[]HandlerFunc`).

- `RouterGroup.Use()` (`routergroup.go:65`) just appends middleware to the group's `Handlers` slice.
- When you register a route via `group.handle()` (`routergroup.go:86`), it calls `combineHandlers()`, which concatenates the group's middleware + the route's handlers into a **single flat slice**, then stores it in the route tree via `engine.addRoute()`. So by request time, middleware and the final handler are indistinguishable — they're all just entries in one ordered chain.

## 2. `ServeHTTP` — entry point (`gin.go:662`)

This satisfies `http.Handler`. Per request it:
- pulls a `*Context` from a `sync.Pool` (object reuse to avoid allocations),
- resets it and attaches the `ResponseWriter` / `Request`,
- calls `handleHTTPRequest(c)`,
- returns the context to the pool.

## 3. `handleHTTPRequest` — the routing lookup (`gin.go:690`)

- Extracts method + path (with optional escaping/slash cleanup).
- Walks `engine.trees` — one radix/prefix tree **per HTTP method** — to find the matching method's tree (`gin.go:709`).
- Calls `root.getValue(rPath, ...)` to match the path, extracting URL params.
- On a match, it assigns the pre-combined chain to the context and kicks it off:

```go
c.handlers = value.handlers   // the flattened middleware+handler chain
c.fullPath  = value.fullPath
c.Next()                      // start executing the chain
```

- If no route matches, it falls through to trailing-slash/fixed-path redirects, then `405` (`allNoMethod`) or `404` (`allNoRoute`) — which are themselves just handler chains run via `serveError → c.Next()`.

## 4. `Context.Next()` — the chain executor (`context.go:188`)

This is the heart of the middleware model:

```go
func (c *Context) Next() {
	c.index++
	for c.index < safeInt8(len(c.handlers)) {
		if c.handlers[c.index] != nil {
			c.handlers[c.index](c)
		}
		c.index++
	}
}
```

- The context holds an **index** into the chain. `Next()` advances the index and invokes each handler in order.
- Middleware controls flow by calling `c.Next()` *inside itself*: code before `c.Next()` runs on the way in, then the nested `Next()` drives the rest of the chain, and code after it runs on the way out. This is what enables the classic "wrap-around" pattern (e.g. the Logger/Recovery middleware timing and recovering around downstream handlers).
- A handler that never calls `c.Next()` still works — the outer `for` loop just continues to the next handler after it returns.

## 5. Aborting (`context.go:207`)

`Abort()` sets `c.index = abortIndex` (a large sentinel value). Because `Next()`'s loop condition is `c.index < len(c.handlers)`, jumping the index past the end stops any pending handlers from running — though the current handler finishes. Helpers like `AbortWithStatusJSON` combine this with a response write.

## Summary of the flow

```
http.Server → Engine.ServeHTTP
  → handleHTTPRequest  (per-method radix tree lookup → chain + params)
    → Context.Next()   (index-driven loop over the flat handler chain)
      → middleware₁ → [calls c.Next()] → middleware₂ → … → final handler
```

The key design insight: **gin has no separate "middleware pipeline."** Middleware and the endpoint handler are merged into one ordered `HandlersChain` at registration time, and `Context.Next()` walks it with a simple integer index — nested `Next()` calls give you the onion-style before/after behavior, and `Abort()` short-circuits by fast-forwarding the index.
