## Overview: three stacked `<canvas>` layers

Excalidraw doesn't render shapes as DOM/SVG — it paints to HTML5 canvases. It uses **separate, stacked canvases** so that frequently-changing UI doesn't force a repaint of the (expensive) element artwork:

| Canvas | Component | Renderer | Holds |
|---|---|---|---|
| **Static** | `components/canvases/StaticCanvas.tsx` | `renderer/staticScene.ts` | the actual elements + grid |
| **Interactive** | `InteractiveCanvas.tsx` | `renderer/interactiveScene.ts` | selection boxes, handles, remote cursors |
| **New-element** | `NewElementCanvas.tsx` | `renderer/renderNewElementScene.ts` | the element currently being drawn |

This split is the core idea: dragging a selection handle only repaints the cheap interactive layer, leaving the static element layer untouched.

## Drawing a single element

`_renderStaticScene` (`staticScene.ts`) iterates `visibleElements`, and for each calls `renderElement(...)` (`packages/element/src/renderElement.ts`), with `save()/restore()` and optional frame clipping around each one. Embeddables/iframes are drawn in a second pass on top, then pending flowchart nodes.

Actual geometry comes from **rough.js**: `_generateElementShape` in `packages/element/src/shape.ts` builds a `Drawable` via a shared `RoughGenerator`, giving the hand-drawn look.

## Two layers of caching (the key to performance)

**1. Shape cache** — `ShapeCache` (`shape.ts:81`), a `WeakMap<element, {shape, theme}>`. `ShapeCache.generateElementShape` returns the cached rough `Drawable` unless missing or theme changed (always regenerated when exporting):

```ts
private static cache = new WeakMap<ExcalidrawElement, { shape; theme }>();
```

**2. Per-element canvas cache** — `elementWithCanvasCache` (`renderElement.ts:603`), another `WeakMap`. Each element is rendered once to its own small offscreen canvas, then blitted onto the static canvas. `generateElementWithCanvas` regenerates only when zoom, theme, bound-text version, image crop, or frame opacity changes — otherwise it reuses the bitmap.

## How updates invalidate caches

Both caches are keyed by the element **object identity** (WeakMap), and mutation is explicit:

- `mutateElement` (`packages/element/src/mutateElement.ts`) applies updates in place, bumps `element.version++` and assigns a fresh `element.versionNonce = randomInteger()`, and **calls `ShapeCache.delete(element)`** when geometry-affecting props (`width`, `height`, `points`, `fileId`) change — which also clears that element's canvas cache.
- `Scene.mutateElement` wraps this and, if the version actually changed and `informMutation` is set, calls `triggerUpdate()`.

## The Scene → render loop

`Scene` (`packages/element/src/Scene.ts`) is the source of truth for elements. `replaceAllElements` / `mutateElement` → `triggerUpdate()`, which sets a new `sceneNonce = randomInteger()` and fires registered callbacks:

```ts
triggerUpdate() {
  this.sceneNonce = randomInteger();
  for (const callback of Array.from(this.callbacks)) callback();
}
```

The `App` component subscribes; a scene change schedules a React re-render. `StaticCanvas` is a `React.memo` whose `areEqual` compares `canvasNonce`, `scale`, the `elementsMap`/`visibleElements` references, and shallow-equal app-state props — so the static layer only re-paints when elements or relevant view state actually change. On each render its `useEffect` calls:

```ts
renderStaticScene({ canvas, rc, elementsMap, visibleElements, appState, renderConfig },
                  isRenderThrottlingEnabled());
```

**Throttling:** `renderStaticScene(config, throttle)` delegates to `renderStaticSceneThrottled = throttleRAF(...)`, coalescing paints to one per animation frame (`requestAnimationFrame`). The interactive scene is throttled the same way. So React decides *whether* to repaint a layer; `throttleRAF` decides *when within the frame*.

## End-to-end flow

```
user action → Scene.mutateElement → mutateElement (version++, versionNonce, ShapeCache.delete)
            → Scene.triggerUpdate (new sceneNonce, fire callbacks)
            → App re-renders → StaticCanvas memo lets it through
            → renderStaticScene (throttleRAF)
            → per element: renderElement → ShapeCache (rough Drawable) + elementWithCanvasCache (bitmap)
            → blit cached element canvases onto the static canvas
```

Unchanged elements keep their cached rough shapes and bitmaps (WeakMap hits); only mutated elements regenerate, and only changed layers repaint — which is what keeps a canvas with thousands of elements interactive.
