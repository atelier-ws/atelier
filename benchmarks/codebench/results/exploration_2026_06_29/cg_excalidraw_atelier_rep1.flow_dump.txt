## The big picture: layered canvases + cached shapes + version-tracked elements

Excalidraw doesn't keep a retained scene-graph that the GPU diffs. Instead it **immediate-mode redraws** onto stacked `<canvas>` layers, but makes that cheap with two caches (shape geometry + viewport culling) and a version system that decides *when* a redraw is even needed.

### Rendering — three stacked canvases

The drawing surface is split into separate canvas layers, each with its own render function and throttling:

| Layer | Component | Render entry | What it draws |
|---|---|---|---|
| **Static** | `StaticCanvas.tsx` | `renderStaticScene()` (`renderer/staticScene.ts`) | The actual shapes, text, images, grid, frames |
| **Interactive** | `InteractiveCanvas.tsx` | `renderInteractiveScene()` (`renderer/interactiveScene.ts`) | Selection boxes, transform handles, remote cursors, scrollbars |
| **New element** | `NewElementCanvas.tsx` | `renderNewElementScene()` | Live preview of the shape currently being drawn |

Splitting them means dragging a selection handle only repaints the lightweight interactive overlay, not every shape on the board.

**Per-element drawing** (`packages/element/src/renderElement.ts` → `renderElement()`) dispatches on element type. Rectangles/ellipses/diamonds/arrows become **rough.js `Drawable`s** (the hand-drawn look) via `RoughGenerator`; freedraw uses perfect-freehand's `getStroke()`; text and images are drawn directly with `drawImage()`. Each draw applies zoom/scroll via `context.scale()`/`translate()`, plus composed opacity (`getRenderOpacity()`) and frame clipping (`frameClip()`).

**Two caches keep redraws cheap:**
- **`ShapeCache`** (`packages/element/src/shape.ts`) — a WeakMap keyed by element, storing the generated rough.js geometry + theme. Generating rough shapes is the expensive part, so it's done once and reused until invalidated.
- **Viewport culling** (`scene/Renderer.ts` → `getRenderableElements()` / `getVisibleCanvasElements()`) — filters to only elements inside the current viewport, memoized, so off-screen shapes cost nothing.

**Scheduling:** redraws are throttled to one per animation frame. `renderStaticSceneThrottled = throttleRAF(_renderStaticScene)` caps the static layer at ~60fps; the interactive layer runs on an `AnimationController` (`renderer/animation.ts`) driven by `requestAnimationFrame` for smooth handle/cursor animation.

### Updating — mutate → version bump → trigger render

1. **Mutation.** Code calls `scene.mutateElement(element, updates)` (wrapping `mutateElement()` in `packages/element/src/mutateElement.ts`). It mutates in place, but only if values actually changed.
2. **Cache invalidation.** If geometry changed (width/height/points/fileId), `mutateElement` calls `ShapeCache.delete(element)` so the rough shape regenerates on next draw. Pure style changes that don't affect geometry can reuse the cached shape.
3. **Version bump.** `bumpVersion()` increments `element.version`, regenerates a random `element.versionNonce`, and sets `element.updated`. This pair is the change-tracking primitive — used for collaboration reconciliation (version compares ordering; nonce breaks ties deterministically) and to detect "did anything change."
4. **Notify.** If the version changed, `Scene.triggerUpdate()` updates `sceneNonce` and fires registered callbacks.
5. **Re-render.** `App.tsx` subscribed via `scene.onUpdate(this.triggerRender)`; `triggerRender()` calls `setState({})` to provoke a React render. React passes fresh `getRenderableElements()` output into the canvas components, whose `useEffect`s call the throttled render functions.

So the cycle is: **`mutateElement` → invalidate ShapeCache + bump version → Scene fires update → App re-renders → throttled canvas redraw, reusing cached shapes for everything that didn't change.**

### Why it's structured this way
- **Layered canvases** isolate frequent cheap repaints (UI/interaction) from rarer expensive ones (shapes).
- **ShapeCache** avoids re-running rough.js geometry generation every frame.
- **Viewport culling** bounds work to what's visible.
- **Version/nonce** doubles as both the "is a redraw needed" signal and the collaboration sync key.
