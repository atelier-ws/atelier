1. **Static Canvas**: Triggered whenever:

## The big picture: three layered canvases

Excalidraw stacks three `<canvas>` elements rather than drawing everything to one, composed in `App.tsx`:

| Canvas | Component | What it draws | Why separate |
|---|---|---|---|
| **Static** | `components/canvases/StaticCanvas.tsx` | Grid + all committed elements + frame outlines | Expensive; only redraws when the scene actually changes |
| **New element** | `components/canvases/NewElementCanvas.tsx` | Just the element currently being drawn | Avoids repainting the whole static scene on every mouse-move while creating a shape |
| **Interactive** | `components/canvases/InteractiveCanvas.tsx` | Selection boxes, transform handles, binding hints, remote cursors | Changes constantly; animation-driven |

Each canvas component is a `React.memo` wrapper whose `useEffect` calls its scene renderer (`renderStaticScene` / `renderNewElementScene` / `renderInteractiveScene`).

## Rendering flow

The render functions live in `packages/excalidraw/renderer/`:

1. **`renderStaticScene`** (`staticScene.ts:491`) → `_renderStaticScene` bootstraps the canvas (clear, set scale/theme), draws the grid, then loops `visibleElements` calling `renderElement(...)` for each.
2. **`renderElement`** (`packages/element/src/renderElement.ts:780`) is a big `switch` on `element.type`. For most shapes it goes through `drawElementOnCanvas` (`renderElement.ts:387`), which fetches the cached shape and hands it to **RoughJS** via `rc.draw(...)` (the hand-drawn look). Freedraw paths fill a `Path2D` directly.
3. **`ShapeCache`** (`packages/element/src/shape.ts:81`) is a **WeakMap** keyed by element reference (+ theme) holding the generated RoughJS `Drawable`. Shapes are regenerated only when missing, so unchanged elements skip the expensive geometry step entirely.

The interactive canvas runs differently: an **`AnimationController`** (`renderer/animation.ts`) drives `renderInteractiveScene` on a `requestAnimationFrame` loop so handles and cursors animate smoothly. Static/new-element renders are batched with **`throttleRAF`** so multiple update calls collapse into one paint per frame.

## Update / mutation flow

Element *state* changes go through the `Scene` and `mutateElement`, separate from rendering:

1. **`mutateElement`** (`packages/element/src/mutateElement.ts:37`) mutates the element object **in place**. It diffs each field, and only if something actually changed it:
   - calls **`ShapeCache.delete(element)`** to invalidate the cached RoughJS shape (so it regenerates next paint),
   - bumps **`element.version`** (sequential counter) and regenerates **`element.versionNonce`** (random) — these drive collaboration reconciliation, not rendering.
2. **`Scene`** (`packages/element/src/Scene.ts`) holds elements as both arrays and `Map`s (`elementsMap`, `nonDeletedElementsMap` for O(1) lookup) and maintains a **`sceneNonce`** — a random int regenerated on every scene update. This is the *render* invalidation key.

## How an update reaches the screen

The link between mutation and repaint:

1. `App` registers `this.scene.onUpdate(this.triggerRender)` (`App.tsx:3147`).
2. After a mutation, `Scene.triggerUpdate()` fires → `triggerRender` (`App.tsx:4655`) calls `this.setState({})`, a React re-render.
3. During render, `Renderer.getRenderableElements` (`scene/Renderer.ts`) computes a **`canvasNonce`** from `sceneNonce`, passed into the canvas components.
4. `StaticCanvas`'s memo comparator (`StaticCanvas.tsx:108`) returns `false` (re-render) only when `canvasNonce`, `scale`, `elementsMap`, or `visibleElements` change — otherwise the canvas is skipped.
5. On re-render, the `useEffect` calls `renderStaticScene`, which repaints — regenerating only the shapes whose `ShapeCache` entry was invalidated.

## The key insight

Two independent identity systems do the work:
- **`version` / `versionNonce`** (per element) → collaboration reconciliation.
- **`sceneNonce` → `canvasNonce`** → React memoization gate deciding whether a canvas repaints.
- **`ShapeCache` WeakMap** → ensures that even when a repaint happens, only *changed* elements pay the cost of regenerating their RoughJS geometry.

So the layered canvases + nonce-based memoization + shape caching together keep redraws cheap: a single element edit invalidates just that element's shape, bumps the scene nonce, and triggers exactly the canvas(es) that need it.
