```swift
func perform(_ request: Request, forRetry isRetrying: Bool = false) {
    rootQueue.async {
        self.mutableState.write { mutableState in
            guard !request.isCancelled else { return }
            guard mutableState.activeRequests.insert(request).inserted || isRetrying else { return }

            self.requestQueue.async {
                // Dispatch based on request type
                switch request {
                case let r as UploadRequest: self.performUploadRequest(r)
                case let r as DataRequest: self.performDataRequest(r)
                case let r as DownloadRequest: self.performDownloadRequest(r)
                case let r as DataStreamRequest: self.performDataStreamRequest(r)
                // ...
                }
            }
        }
    }
}
```

- Adds request to `mutableState.activeRequests` (a `Set<Request>`)
- Dispatches work to `requestQueue` (a serial queue targeting `rootQueue`)

Retry logic in `Session.retryResult(for:dueTo:completion:)`:
```swift
public func retryResult(for request: Request, dueTo error: AFError, 
                        completion: @escaping @Sendable (RetryResult) -> Void) {
    guard let retrier = retrier(for: request) else {
        rootQueue.async { completion(.doNotRetry) }
        return
    }

    retrier.retry(request, for: self, dueTo: error) { retryResult in
        self.rootQueue.async {
            guard let retryResultError = retryResult.error else { completion(retryResult); return }

            let retryError = AFError.requestRetryFailed(retryError: retryResultError, originalError: error)
            completion(.doNotRetryWithError(retryError))
        }
    }
}
```

The retrier receives the composed `RequestInterceptor` (session-level + request-level) and can return:
- `.retry` / `.retryWithDelay(TimeInterval)` → triggers `Session.retryRequest(_:withDelay:)`
- `.doNotRetry` → proceeds to `finish()`
- `.doNotRetryWithError(Error)` → wraps error and proceeds to `finish(error:)`

#### Retry Execution

```swift
public func retryRequest(_ request: Request, withDelay timeDelay: TimeInterval?) {
    rootQueue.async {
        let retry: @Sendable () -> Void = {
            guard !request.isCancelled else { return }

            request.prepareForRetry()  // Increments retryCount, calls reset()
            self.perform(request, forRetry: true)  // Re-enqueues request to requestQueue
        }

        if let retryDelay = timeDelay {
            self.rootQueue.after(retryDelay) { retry() }
        } else {
            retry()
        }
    }
}
```

Retries re-enter the SEND phase: a new `URLSessionTask` is created and all validations run again.

#### processNextResponseSerializer(): Sequential Execution

```swift
func processNextResponseSerializer() {
    let executeOutside: (() -> Void)? = mutableState.write { mutableState in
        guard !mutableState.isResponseSerializerEnqueued else { return nil }

        let responseSerializerIndex = mutableState.responseSerializerCompletions.count
        let isAvailableSerializer = responseSerializerIndex < mutableState.responseSerializers.count
        let responseSerializer = isAvailableSerializer ? mutableState.responseSerializers[responseSerializerIndex] : nil

        if let responseSerializer {
            mutableState.isResponseSerializerEnqueued = true
            return { self.serializationQueue.async { responseSerializer() } }
        } else {
            let completions = mutableState.responseSerializerCompletions
            mutableState.responseSerializers.removeAll()
            mutableState.responseSerializerCompletions.removeAll()

            if mutableState.state.canTransitionTo(.finished) {
                mutableState.state = .finished
            }

            mutableState.responseSerializerProcessingFinished = true
            mutableState.isFinishing = false

            return {
                completions.forEach { $0() }
                self.cleanup()
            }
        }
    }

    executeOutside?()
}
```

Response serializers (response handlers like `.response()`, `.responseJSON()`, etc.) are stored in `mutableState.responseSerializers` and executed sequentially on `serializationQueue`. Each serializer processes the validation error (if any) in `request.error`, computes its result, and enqueues a completion closure. When all serializers complete, all completion closures are executed and the request is cleaned up.

## 1. Build

**Entry point** — `Session.request(...)` in `Source/Core/Session.swift` is a convenience method taking a URL, method, parameters, encoding, headers, and an optional interceptor. It wraps everything into an internal `URLRequestConvertible` and returns a `DataRequest`.

- **`URLRequestConvertible`** (`URLConvertible+URLRequestConvertible.swift`) is the core abstraction — its `asURLRequest()` produces the actual `URLRequest`. Two internal implementations: `RequestConvertible` (dictionary params + `ParameterEncoding`) and `RequestEncodableConvertible` (`Encodable` params + `ParameterEncoder`).
- **Parameter encoding** has two paths:
  - *Legacy* `ParameterEncoding` (`URLEncoding`, `JSONEncoding`) — encodes `[String: Any]` into query string or body.
  - *Modern* `ParameterEncoder` (`URLEncodedFormParameterEncoder`, `JSONParameterEncoder`) — type-safe `Codable` encoding.
- **`DataRequest`** is created via `Session.request(_:interceptor:...)` and held lazily. Setup is deferred until `resume()`, then **`performSetupOperations(for:convertible:)`** runs: it calls `asURLRequest()`, validates the request, then applies any **`RequestAdapter`** (from a `RequestInterceptor`) asynchronously on `requestQueue` to mutate it — e.g., inject auth headers — before task creation.

## 2. Send

- **`Session.perform(_:forRetry:)`** adds the request to `activeRequests` on `rootQueue`, then dispatches to `requestQueue`, switching on concrete type (`performDataRequest`, `performUploadRequest`, etc.).
- **`Request.task(for:using:)`** — overridden in `DataRequest.swift` — produces the `URLSessionTask` (`session.dataTask(with:)`). The pairing is stored in **`RequestTaskMap`** (`RequestTaskMap.swift`), a bidirectional `Request ↔ URLSessionTask` map that also tracks whether completion and metrics have both arrived before disassociating.
- **Queues**: `rootQueue` (serial, all state mutation), `requestQueue` and `serializationQueue` (both target rootQueue), plus the `URLSession`'s `delegateQueue` (an `OperationQueue` backed by rootQueue).
- **`SessionDelegate`** (`SessionDelegate.swift`) implements the URLSession delegate protocols: `didReceive response` / `didReceive data` accumulate the response, `didReceive challenge` handles server-trust pinning and credential auth, and `didCompleteWithError` fires the completion chain (mapping errors to `AFError`).

## 3. Validate

- **`validate()`** on `DataRequest` (`Source/Features/Validation.swift`) composes status-code (`200..<300`) and content-type (from the `Accept` header, default `*/*`) validators. You can also pass a custom closure of type `Validation = (URLRequest?, HTTPURLResponse, Data?) -> ValidationResult`, where `ValidationResult = Result<Void, Error>`.
- Validators are stored as closures and run in **`Request.didCompleteTask(_:with:)`** on `underlyingQueue`. A failure sets `request.error` to an **`AFError.responseValidationFailed`** whose `ResponseValidationFailureReason` is one of `.unacceptableStatusCode`, `.unacceptableContentType`, `.missingContentType`, `.customValidationFailed`, etc. (`AFError.swift`).
- After validators, **`retryOrFinish(error:)`** consults the `RequestRetrier`. It can return `.retry` / `.retryWithDelay` — re-entering the send phase via `Session.retryRequest` with a fresh task and re-run validation — or `.doNotRetry` → `finish()`.
- **`finish()` → `processNextResponseSerializer()`** then runs response serializers sequentially on `serializationQueue`; each sees any validation error in `request.error`, and completion handlers fire once all serializers are done.

**One-line mental model:** `request()` builds a `URLRequestConvertible` → adapter mutates it → a `URLSessionTask` is created and mapped → `SessionDelegate` collects the response → validators set `AFError` on failure → retrier decides retry-or-finish → serializers deliver the result.
