## How the extension host talks to the main process (VS Code model)

The extension host is a **separate process** from the main (renderer/window) process. They never share memory or call each other directly — all communication goes over a **message-passing RPC channel**.

**The pieces:**

1. **Separate process.** Extensions run in a dedicated Node.js process (the "extension host"), isolated from the UI. This keeps a misbehaving extension from freezing the editor's rendering and keeps extensions off the UI thread entirely.

2. **A duplex byte stream connects them.** Depending on the setup, that transport is a socket, a Node `MessagePort`/IPC channel, or a WebSocket (for remote/web scenarios). Everything below is layered on top of this single stream.

3. **RPC protocol on top of the stream.** VS Code uses an internal RPC layer (`RPCProtocol` over an `IMessagePassingProtocol`). Messages are length-prefixed and serialized (JSON, with special handling for `Buffer`/`VSBuffer` and `URI` objects). Each message is a request, a reply, or an event/cancellation.

4. **Proxies and "shapes."** Both sides agree on a set of typed interfaces declared in `extHost.protocol.ts`:
   - `MainThreadXxx` services live in the main process.
   - `ExtHostXxx` services live in the extension host.
   - Each side gets **proxy objects** for the other side's services (`getProxy(...)`). Calling a method on a proxy serializes the call, ships it over the stream, runs it on the far side, and returns a `Promise` with the result. So what looks like a normal async method call is actually a marshalled round-trip.

5. **The `vscode` API is a facade.** When an extension calls `vscode.window.showInformationMessage(...)`, that runs in the extension host, which invokes a `MainThread*` proxy; the real work happens in the main process, and the result comes back over the same channel. Async-only by design — there are no synchronous cross-process calls.

**Implications you'll feel as an extension author:**
- Every API that crosses the boundary is `Promise`-based.
- You can't pass live object references across — only serializable data (hence `URI`, `Position`, etc. are plain data types).
- Heavy/blocking work in an extension can't hang the UI, but a chatty extension can still create latency from RPC round-trips.
