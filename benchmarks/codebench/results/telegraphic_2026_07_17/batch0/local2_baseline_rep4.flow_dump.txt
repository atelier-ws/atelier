## What's wrong

**Unit mismatch.** JWT's `exp` claim is in **seconds** since the Unix epoch (per RFC 7519 — it's a *NumericDate*). But `Date.now()` returns **milliseconds**.

So you're comparing two numbers that are off by a factor of 1000:

```js
// exp        ≈ 1,752,700,000        (seconds)
// Date.now() ≈ 1,752,700,000,000    (milliseconds)
```

`Date.now()` is always ~1000× larger. If your check is written the intuitive way:

```js
if (payload.exp < Date.now()) {   // "is exp in the past?"
  throw new Error('expired');
}
```

`exp` (seconds) is *always* less than `Date.now()` (ms), so this would actually reject **every** token. If instead you wrote it the other direction, or `Date.now() < exp`, the comparison is always false and **every expired token passes** — which matches your symptom.

## The fix

Convert to the same unit. Divide `Date.now()` by 1000:

```js
const nowSec = Math.floor(Date.now() / 1000);
if (payload.exp < nowSec) {
  return res.status(401).json({ error: 'Token expired' });
}
```

## Better fix: don't hand-roll the check

If you're using `jsonwebtoken`, `jwt.verify()` already validates `exp` for you (in the correct units) and throws a `TokenExpiredError`. If expired tokens are getting through, you're probably using `jwt.decode()` (which does **no** verification) instead of `jwt.verify()`:

```js
const jwt = require('jsonwebtoken');

function requireAuth(req, res, next) {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) return res.status(401).json({ error: 'No token' });

  try {
    req.user = jwt.verify(token, process.env.JWT_SECRET); // checks sig + exp
    next();
  } catch (err) {
    // TokenExpiredError, JsonWebTokenError, etc.
    return res.status(401).json({ error: 'Invalid or expired token' });
  }
}
```

This also gives you signature verification and a small clock-skew tolerance (`clockTolerance` option) for free.

**Two things to double-check:**
1. If you manually set `exp` when signing, make sure that's also in seconds (`Math.floor(Date.now()/1000) + 3600`), not milliseconds — otherwise you'd bake in tokens that "expire" ~55,000 years from now.
2. Make sure the comparison actually **rejects** (returns/throws) rather than just logging.
