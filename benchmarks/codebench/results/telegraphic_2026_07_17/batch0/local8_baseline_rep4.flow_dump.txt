[assistant] Done. `Dockerfile` created. Key size-minimizing decisions:

**Three stages, so devDependencies and source never reach the final image:**
- **`build`** — installs all deps (`npm ci`) and compiles TypeScript → `dist/`.
- **`deps`** — a separate clean install of *only* production deps (`npm ci --omit=dev`). Kept separate from `build` so the TypeScript compiler and `@types/*` are never copied forward.
- **`runtime`** — copies just three things: prod `node_modules`, compiled `dist/`, and `package.json`.

**Other choices:**
- `node:22-alpine` base — small footprint vs. the Debian-based default.
- `npm ci` (not `npm install`) — reproducible installs from `package-lock.json`, and it's designed for CI/containers.
- Manifests copied before source so the dependency layer stays cached until `package.json`/lockfile change.
- `USER node` + `--chown=node:node` — runs unprivileged; the `node` user ships with the official image.
- `NODE_ENV=production`.

**You may need to adjust:**
- **Build script** — assumes `npm run build` exists and `tsconfig.json` emits to `./dist`. If your `outDir` or entrypoint differs, update the `COPY --from=build /app/dist` and `CMD ["node", "dist/index.js"]` lines.
- **Port** — `EXPOSE 3000`; change if your app listens elsewhere.
