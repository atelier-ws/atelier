**What it does:**
- **Catches render errors** via `getDerivedStateFromError` (switches to fallback) and `componentDidCatch` (side effects/logging).
- **Fallback UI with retry** — default styled alert showing the error message plus a "Try again" button; `retry()` clears the error state so the subtree re-renders. Overridable via a `fallback(error, retry)` render prop.
- **Logs details** — `console.error` with the error and `componentStack`, plus an optional `onError` callback for wiring to an external service (Sentry, etc.).

**Usage:**
```tsx
import { ErrorBoundary } from "./components/ErrorBoundary";

<ErrorBoundary onError={(err, info) => reportToSentry(err, info)}>
  <YourApp />
</ErrorBoundary>
```

One caveat worth knowing: error boundaries only catch errors during render/lifecycle/constructors of descendants — not event handlers, async code, or SSR. Those need `try/catch`.
