## Post-Benchmark Fixes (Not in Original Plans)

Two bugs fixed after v1.0 shipped (commit e587a86, cc d882f0d):

1. **Stripped internal MCP fields** — `cache_hit`, `total_tokens`, `tokens_saved`, `symbol_id`, `edges`, `repo_id`, `start_byte`, `end_byte`, `content_hash`, `score`, `provenance_breakdown` removed from LLM-facing payloads. Reduces token bloat without losing useful information.

2. **`code:` op prefix aliases** — External benchmarks use `code:callers`, `code:cache_status` etc. Atelier's Pydantic `Literal` rejected these → all ops scored 0. Added `_tool_code_alias_handler` to normalize prefix before dispatch.

These fixes post-date the v1.0 milestone close but materially affect the external benchmark scores described in the user's audit brief.
