## Methodology

**Sessions analysed:** Real Claude Code session exports (`.jsonl` format). Synthetic sessions and sub-agent context flushes excluded.

**Corpus source:** `/home/pankaj/.atelier/benchmarks/savings`

**Pricing assumptions:** Per-model input/output rates from `atelier/infra/runtime/pricing.py`. Rates as of report generation.

**Routing benchmark:** Atelier's `ModelRouter` is run per-turn on each session. Cost delta = actual model cost - recommended model cost per turn. Only positive deltas are counted (never inflated by assuming the model was worse).

**Compaction benchmark:** Atelier estimates freed tokens using its context compressor model, compared to the measured native `/compact` output. Delta reflects the *additional* savings only.

**Quality benchmarks:** Proxy signals from real session data (no LLM replay for quality). Routing quality: `risk = 0.35 x tool_risk + 0.20 x output_complexity + 0.25 x model_error + 0.20 x retry_signal`. Compact quality: `retention_score = 1 - clamp(error_drift x 2 + extra_read_rate x 0.5, 0, 1)`.

**Replay benchmark:** True counterfactual via Claude CLI. Haiku is called with reconstructed session context and asked to predict the next tool call. No API key required.

**Exclusions:** Sessions with fewer than 80,000 peak context tokens excluded from compact benchmarks. Sessions already using cheap models excluded from routing savings count.
