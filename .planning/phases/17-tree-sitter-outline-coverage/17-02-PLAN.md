<objective>
Add the three descent-dependent language configs — `sql`, `yaml`, `json` — to `_LANG_CONFIG`. These
languages have no top-level declarations: their structure is buried inside transparent wrapper nodes
(`statement` for SQL; `stream→document→block_node→block_mapping` for YAML; `document→object` for JSON),
so they only become reachable thanks to the `unwrap` descent and `keep_first_line` emit added in Plan
17-01.

Purpose: Complete DLS-OUTLINE coverage. SQL exposes schema-level constructs (tables/views/indexes/
functions); YAML exposes top-level document keys; JSON exposes top-level object structure when the 25%
guard justifies it (and degrades cleanly to generic/full on small/flat files — the designed
DLS-OUTLINE-05 behavior).

Output: Three new `_LANG_CONFIG` entries and three new test files. The full outline regression set stays
green.
</objective>
