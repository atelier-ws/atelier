|----------|-------------|
| MCP caller → code-intel wrappers | Untrusted search-derived symbol queries reach semantic tools through public MCP handlers |
| Code-intel engine → local index/search stores | Escalation queries must stay budgeted and workspace-scoped even when indexes are stale or partial |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-01-02-02 | Information Disclosure | `CodeContextEngine` result packing | mitigate | Preserve existing path/scope/budget controls and avoid new eager full-file payloads when escalating from search. |
| T-01-02-03 | Denial of Service | auto-index / semantic exploration | mitigate | Keep bounded `budget_tokens`, limited relationship fan-out, and local fallback behavior so stale or missing indexes do not stall the loop. |
| T-01-02-SC | Tampering | npm/pip/cargo installs | accept | No package-manager changes in this plan. |
</threat_model>
