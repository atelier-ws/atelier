# Phase 2: Plan 07 - Read-Only Structural Minification

## Phase Goal

**As a** terminal coding agent user, **I want to** read files through a more aggressive minified reader path, **so that** explore-time token spend drops (the read half of Eval's VFS) without ever mutating what the implementer sees.

<objective>
Deliver the read-side token win from Eval's Virtual File System inside Atelier's existing reader-profile minifier, without the edit/write round-trip (deferred to v2 TOOL-01).

Purpose: satisfy EXEC-14 and reinforce GRND-03/INTL-03 by reducing explore-read tokens while preserving byte-exact reads for writer/execute paths and the existing minification telemetry contract.

Output: an intra-line whitespace collapse for formatter-safe languages, wired into the existing reader read handler, with safety and token-reduction coverage.
</objective>

| Boundary | Description |
|----------|-------------|
| file body -> reader minifier | Source content is transformed before entering agent context |
| reader minifier -> writer/execute reads | A leaked aggressive transform onto the writer path would corrupt edit matching |

| T-02-SC | Tampering | package installs | mitigate | No new package installs in this plan |
