<objective>
Harden the MCP stdio smoke test so it strictly rejects any non-protocol bytes on the
server's stdout. Today the test (L78-83) swallows unparseable lines in a bare
`try/except: pass`, so a stray `print()` would slip through and the test would still
pass. This plan replaces the lenient parse with a strict per-line `json.loads` that
asserts every non-empty stdout line is a JSON object — turning the smoke test into a
real framing-integrity gate (QBL-LOG-04) BEFORE the conversion plans (24-02/24-03)
rely on it.

Purpose: Establish the QBL-LOG-04 gate first so later conversions are verifiably safe.
Output: A strict, runnable `test_mcp_stdio_smoke.py`.
</objective>

<task type="auto">
  <name>Task 1: Harden stdio smoke parse to strict JSON-object assertion</name>
  <files>tests/gateway/test_mcp_stdio_smoke.py</files>
  <action>
Replace the lenient stdout parse loop (currently around L78-83 — the
`try: msg = json.loads(line) ... except Exception: pass` block) with a strict parse
modeled on `test_mcp_jsonrpc_e2e.py` L306. For every non-empty stdout line, call
`json.loads(line)` with NO try/except, then `assert isinstance(msg, dict)` with a
message that echoes the offending line (e.g. `f"non-protocol stdout line: {line!r}"`),
and only then index `msg["id"]` into the responses map. The goal: any stray non-JSON
or non-object line now fails the test rather than being silently dropped. Do NOT change
how the subprocess is launched, what requests are sent, or the existing response
assertions — only the stdout parsing strictness. Keep the existing
`@pytest.mark.slow` marker; do not remove it.
  </action>
  <verify>
    <automated>uv run pytest tests/gateway/test_mcp_stdio_smoke.py -m "" -q</automated>
  </verify>
  <done>
The smoke test passes with `-m ""`; the stdout loop contains a bare `json.loads(line)`
(no surrounding try/except) and an `isinstance(msg, dict)` assertion; the
`try/except: pass` swallow is gone.
  </done>
