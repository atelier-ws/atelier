<objective>
Convert the import-progress `print()` calls in the 9 session-parser modules to module
loggers (per QBL-LOG-02), and pair the conversion with a minimal INFO-level stderr
`StreamHandler` attached on the CLI import-command path so the user still sees progress
— on stderr, never stdout (per QBL-LOG-03). Without that handler the converted
`logger.info()` calls would vanish (the CLI's root logger defaults to WARNING with no
handler — RESEARCH Pitfall 1). This is NOT CLI decomposition (Phase 25 owns that); it
is a single small handler attachment plus a test.

Purpose: Remove the largest cluster of non-CLI prints while keeping import UX intact.
Output: Logger-based parsers, a stderr progress handler, and a CLI progress test.
</objective>
