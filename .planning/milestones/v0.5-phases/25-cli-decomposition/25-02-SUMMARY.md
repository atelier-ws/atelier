# Phase 25 Plan 02: Extract OpenMemory/Letta Commands + Lifecycle Service Summary

Extracted the `openmemory` and `letta` command groups into thin
`cli/commands/` modules and relocated their non-CLI business logic (OpenMemory
git checkout, env-file generation, `docker compose` orchestration, `make`
invocation, and the path layout) into a new
`gateway/integrations/openmemory_lifecycle.py` service — with the live help
tree proven **byte-identical** before/after (sha256
`42a2fed…`) and generated env/compose output preserved verbatim.

## What Was Built

### Task 1 — `gateway/integrations/openmemory_lifecycle.py` (QBL-CLI-03)
- NEW sibling of the MCP-over-HTTP client `openmemory.py` (per RESEARCH A2 — the
  client talks to a running server; this module manages the server lifecycle).
- Moved verbatim from `app.py`: `letta_compose_file`, `run_compose`,
  `project_root`, the OpenMemory path layout (`openmemory_dir`,
  `openmemory_checkout_dir`, `openmemory_workdir`, `openmemory_service_env_path`,
  `openmemory_api_env_path`, `openmemory_ui_env_path`, `openmemory_log_path`,
  `mcp_dir`, `mcp_log_path`), and the lifecycle ops (`ensure_service_env`,
  `ensure_checkout`, `write_env_files`, `run_make`).
- Public names (leading underscore dropped). Style matches the existing
  integrations module: `from __future__ import annotations`, module-level
  `logger`, stdlib `subprocess`/`contextlib`/`os`, env via `os.environ.get`.
- `ensure_checkout` now raises `RuntimeError` (not `click.ClickException`) — the
  CLI wrapper converts. All other infra behavior is byte-identical.

### Task 2 — `cli/commands/openmemory.py` + `cli/commands/letta.py` (QBL-CLI-02)
- `letta.py` exports `letta_group` (`@click.group("letta")`) with `up`/`down`/
  `status`/`reset` — docstrings, the `--yes` flag, defaults, and help strings
  copied VERBATIM. Callbacks lazily import `run_compose` and call it.
- `openmemory.py` exports `openmemory_group` with `up`/`down`/`status`/`logs`.
  The `git`/`docker`/`make` `shutil.which` preflight, the
  `OPENAI_API_KEY` check, `ctx.obj["root"]` access, `click.echo`, and
  `subprocess.run(..., check=False)` for `status`/`logs` all stay in the
  command. `ensure_checkout`'s `RuntimeError` is converted to
  `click.ClickException(str(exc))` (identical message).
- Both defined as standalone `click.Group`s (not `@cli.group`) to avoid an
  import cycle (Pattern 1); registered in `commands/__init__.py`'s `register(cli)`
  via `add_command`, each wrapped in `try/except ModuleNotFoundError` mirroring
  the substrate style.

### Task 3 — `app.py` cleanup (QBL-CLI-01 progress)
- Deleted the `letta_group`/`openmemory_group` definitions and all 16 moved
  path/lifecycle helpers from `app.py`.
- Removed the now-unused `import urllib.request` (only `letta_status` used it;
  it moved to `commands/letta.py`).
- Re-imported the six helpers still referenced by not-yet-extracted commands
  (systemd/background/stack slices) back under their original underscore names:
  `_run_compose`, `_project_root`, `_openmemory_service_env_path`,
  `_openmemory_log_path`, `_mcp_log_path`, `_ensure_openmemory_service_env`.

### [Documented - QBL-CLI-04 boundary] `ensure_checkout` exception type changed (behavior preserved)
- The lifecycle `ensure_checkout` raises `RuntimeError` instead of
  `click.ClickException` (plan-mandated: infra raises domain errors, CLI
  converts). The `openmemory_up` wrapper catches `RuntimeError` and re-raises
  `click.ClickException(str(exc))`, so the user-facing message
  (`"OpenMemory checkout is missing …"`) and exit behavior are unchanged. Not a
  help/output difference — flagged here for completeness.
