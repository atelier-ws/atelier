<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → repo | The nightly workflow executes with a `GITHUB_TOKEN`; over-privileged scope is the main risk in a CI/config phase. |
| CI runner → dependency registry | `uv sync` resolves dependencies; un-pinned resolution can pull drifted/malicious versions. |
| Lint config → future contributors | An over-broad `ignore` would silently re-admit blind-except/print debt across the whole repo. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01 | Tampering | `pyproject.toml` lint ignores | mitigate | Use scoped `[tool.ruff.lint.per-file-ignores]` keyed to specific files; never add BLE001/T201 to top-level `ignore`. Regression probe in Task 1 proves new debt in a clean file fails lint. |
| T-22-02 | Tampering | `nightly-coverage.yml` dependency install | mitigate | `uv sync --frozen --group dev` pins against `uv.lock`; no floating versions. Pinned action SHAs/tags (`@v4`/`@v5`) copied from existing workflows. |
| T-22-03 | Elevation of Privilege | `nightly-coverage.yml` workflow token | mitigate | `permissions: contents: read` (least privilege) — the job only reads code and runs tests; no write scopes granted. |
| T-22-04 | Information Disclosure | Coverage report artifacts | accept | `make test-full` emits `term-missing` to job logs only; no HTML upload, no secrets in coverage output. Low value, public benchmark repo. |
| T-22-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed this phase (RESEARCH.md Package Legitimacy Audit: N/A). `uv sync --frozen` only restores already-pinned deps. No `[ASSUMED]`/`[SUS]` packages → no legitimacy checkpoint required. |
</threat_model>
